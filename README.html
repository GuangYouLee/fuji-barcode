Scan Modes

The operator selects a mode before scanning. Both paths converge at Step 3 (script fetch).

🟠 Log ID Mode

  • Scanned value = a log identifier
  • App queries SQLite table barcode_recipe_mappings
  • DB returns recipe_name → continues to script fetch

🔵 Recipe Mode

  • Scanned value = the recipe name directly
  • No database lookup needed
  • Proceeds straight to script fetch
🎯

Script Matching Logic

Implemented in Helpers/RecipeScriptResolver.cs. Rules applied in priority order:

Priority Rules

  1. Exact match on script name → wins immediately.
  2. Names starting with recipeName_ are candidates.
  3. Highest _Vx.y version among candidates wins.
  4. Ambiguous non-versioned tie → error, no guess.

Example

Recipe name scanned:

group_script1

Matched script selected:

group_script1_202605261129_V1.0
Versioned match wins

UI Screens

Main Screen (Operator)

The primary operator-facing window. Built for speed — scan in, result out.

  • Single scan input box, auto-focused on load
  • Mode toggle: Log ID / Recipe
  • Submit via Enter or button
  • Input clears automatically after each scan
  • Guarded by IsBusy — no duplicate triggers
  • Status text shows result inline

Logic: ViewModels/MainWindowViewModel.cs

Admin Screen (Mappings CRUD)

Maintains the barcode_recipe_mappings table from within the app.

  • Create: enter Log ID + Recipe → Save
  • Read: click Refresh to load mappings list
  • Update: select row → edit → Save
  • Delete: select row or enter ID → Delete
  • Save & Delete both require confirmation dialog

Logic: ViewModels/AdminWindowViewModel.cs · Views/AdminWindow.axaml

Database (SQLite)

Uses embedded SQLite via Microsoft.Data.Sqlite — no external database server required. Schema creation and data access: Services/BarcodeLookupService.cs.

The database file is created automatically in the per-user local app-data folder, alongside user-preferences.json. On Windows that is C:\Users\<username>\AppData\Local\fuji-barcode\barcode.db. On Linux it resolves through .NET's LocalApplicationData path.

Table: barcode_recipe_mappings

  • log_id — scanned barcode value (primary key)
  • recipe_name — mapped recipe for script lookup
  • updated_at — last-modified timestamp (ISO 8601 text)

RPA Engine Integration

Communicates with rpa-engine over HTTP. Client: Services/RpaEngineClient.cs.

Config (appsettings.default.json)

appsettings.json 
{
  "RpaEngine": {
    "BaseUrl":    "http://localhost:5000",
    "ApiKey":     "",
    "TargetName": ""
  }
}

API Calls Made

  • GET /api/scripts — returns all top-level folders and standalone .json files from the engine's scripts directory. Used by the app to build the list for matching.
  • POST /run/{scriptName} — triggers asynchronous script execution. Engine responds immediately with HTTP 202 Accepted ({"status":"started"}); it does not wait for the script to finish.
ApiKey → optional X-API-Key header TargetName → optional ?target=… param

Typical Operator Flow

Step-by-step comparison for both scan modes from the operator's perspective.

# 🟠 Log ID Mode 🔵 Recipe Mode
1 Leave mode set to Log ID Switch mode to Recipe
2 Scanner reads barcode label Scanner reads recipe name label
3 App queries log_id → recipe_name in SQLite DB lookup skipped
4 Both paths: app resolves the best matching engine script
5 Both paths: app calls POST /run/{scriptName} on rpa-engine
6 Both paths: status text updates with result (success or error)

Install

Windows

Use the MSI build script, then install the generated package.

PowerShell 
powershell -ExecutionPolicy Bypass -File .\packaging\windows\build-msi.ps1

Installer output: artifacts/installers/fuji-barcode-win-x64.msi

  1. Double-click fuji-barcode-win-x64.msi
  2. Follow the Windows Installer prompts
  3. Launch Fuji Barcode from the desktop or Start Menu shortcut
  4. Edit C:\Users\<username>\AppData\Local\fuji-barcode\appsettings.json if the rpa-engine settings need to change
Self-contained Windows x64 MSI

Linux

There is no Linux installer package in this repo now. Publish and copy the app folder manually.

bash 
dotnet publish ./fuji-barcode.csproj -c Release -r linux-x64 --self-contained true -o ./artifacts/publish/linux-x64
  1. Copy ./artifacts/publish/linux-x64/ to the target Linux machine
  2. Make the binary executable if needed: chmod +x ./fuji-barcode
  3. Run the app from that folder: ./fuji-barcode
  4. Edit ~/.local/share/fuji-barcode/appsettings.json if the rpa-engine settings need to change
Manual Linux install
